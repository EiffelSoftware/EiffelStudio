Archives

Icon in the metric tool is provided to handle archives. This operation is really specific. It allows comparison between system scopes.

Indeed, it is actually easy to compare smaller scopes (such as features, classes and clusters) one to another among the same project. But this is different for systems since one project contains one and only one system. That is why archives have been created, to store measures for a given system when it is loaded and retrieve them later on when another system is being loaded.

The metric tool allows two kinds of facilities concerning archives:

  • Creation and update for the loaded system
  • Comparison to another system whose measures have been archived

 

By clicking on , the following dialog pops-up:

Which allows archive creation update and comparison. for more details see section: handling archives 

The metric tool creates automatically a sub folder named "Metrics" in the current location of the loaded project. It is advised, but not compulsory, to save created archives in that folder, since it has been provided for metric needs.
