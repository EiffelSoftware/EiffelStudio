Layout Constructor

The layout constructor contains a representation of the objects you have defined in your system.

Each item in the tree represents one object. The screenshot above represents the following Vision2 interface definition - An EV_TITLED_WINDOW containing one child, of type EV_VERTICAL_BOX. Within the vertical box there is an EV_HORIZONTAL_BOX and an EV_LIST. The list is empty, but the horizontal box contains three different objects of type EV_BUTTON.

Note: The text of each tree item corresponds to the type of the object. BUTTON corresponds to EV_BUTTON, LIST to EV_LIST etc etc.

If you have just started a new project, then the layout constructor will only contain one tree item, TITLED_WINDOW.
