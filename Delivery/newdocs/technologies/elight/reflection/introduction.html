1. Introduction

ISE reflection system is composed of five components interacting with each other: `ISE.Reflection.Emitter.exe', `ISE.Reflection.EiffelComponents.dll', `ISE.Reflection.Support.dll', `ISE.Reflection.CodeGenerator.dll' and `ISE.Reflection.ReflectionInterface.dll'.

  • `ISE.Reflection.Emitter.exe' enables you to generate XML files and Eiffel classes from a .NET assembly. This relies on the three other components described below. You can call `ISE.Reflection.Emitter.exe' directly.


  • `ISE.Reflection.EiffelComponents.dll' is used internally by `ISE.Reflection.Emitter.exe', `ISE.Reflection.CodeGenerator.dll' and `ISE.Reflection.ReflectionInterface.dll'. It provides a representation of an Eiffel class, an Eiffel feature and an Eiffel assembly.


  • `ISE.Reflection.Support.dll' provides a feature `hash_value', which computes hash value of a string by using the MD5 hash algorithm. This is used by the database to generate folder name for an assembly, and by the generators to find file location.


  • `ISE.Reflection.CodeGenerator.dll' is used by `ISE.Reflection.Emitter.exe' to generate XML files and Eiffel classes from Eiffel components. It is also used by `ISE.Reflection.ReflectionInterface.dll' to retrieve Eiffel components from XML files.


  • `ISE.Reflection.ReflectionInterface.dll' is aimed at connecting .NET names with corresponding Eiffel names by using XML files. It is a kind of database where you put XML files describing .NET types and you retrieve a user-friendlier type description. (Only shared assemblies can be stored in this database.) You can use it directly to retrieve information from the database.

Since all other components rely on it, this specification describes `ISE.Reflection.EiffelComponents.dll' first.
