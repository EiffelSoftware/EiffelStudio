Handling archives

The archive file for a system contains all the values of measures resulting from evaluating all metrics over the "system" scope when evaluation is possible. It also contains the definitions of non elementary metrics (the one user has defined or imported) in order to be sure of what is being measured.

The archive files are entitled to achieve comparisons between systems. And then, user can compare systems to evaluate their own projects.

When clicking on in the metric tool, the following interface will help you in:

  • Creating an archive
  • Updating an existing archive
  • Comparing to an archive

 

An archive can also be considered as a scope even if it is not a sub-element of the loaded system users are working on. Indeed, an archive contains measures, for another system than the current one,  previously calculated and registered that could be retrieved. Consequently, evaluating a metric over an archive is nothing but retrieving the corresponding recorded measure.
