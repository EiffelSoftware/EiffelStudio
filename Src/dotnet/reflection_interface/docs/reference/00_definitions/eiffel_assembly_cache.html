The Eiffel Assembly Cache

The Eiffel Assembly Cache is a repository where the compiler saves the required information to be able to associate Eiffel names with .NET names.

It keeps tracks of the changes you make when editing types via the Type View: the Eiffel Assembly Cache is updated (i.e. old names replaced by new ones) after the Assembly Manager has checked that the changes are consistent.

The Assembly Manager is notified whenever the Eiffel Assembly Cache is modified, so that it always displays the relevant names.

Note: The Eiffel Assembly Cache is self-sufficient: When you launch the Eiffel code generation from the Assembly Manager (via the Eiffel Generation Dialog), the generator finds all the information it needs in the Eiffel Assembly Cache. There is no need to explore the assembly metadata anymore, which results in a noteworthy gain of time.

The Eiffel Assembly Cache is located in $ISE_EIFFEL\dotnet\assemblies, where $ISE_EIFFEL is the path to your Eiffel delivery. If you explore it, you will see that information is saved in separate folders, each folder corresponding to one imported assembly.
