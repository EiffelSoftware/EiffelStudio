The type view


The type view is an editable short form of an Eiffel class. At the top of the window, the name, version, culture and public key of the assembly the edited type belongs to is repeated.
In the panel below, you will notice that some words appear with a colored background. These are the words, which are editable. It includes the the name of each local feature and its arguments names (if any), except for the creation routine.

Note: Only local features (i.e. first defined in the currently edited class) and its arguments names are editable. If you want to rename a feature, which is inherited (or an argument whose related feature is inherited) you have to switch to the parent class to change the feature name (or argument name).


Facilities provided by the type view:

  • Rename a class feature
    When you open the type view (by clicking on the type Eiffel name in the assembly view), the text field with first class feature name is automatically selected. You just have to type the new feature name.

  • Rename a feature argument
    To rename a feature argument, just give the focus to the text field you want to edit (by clicking with the mouse) and type the new argument name.

Note that your changes will only be taken into account when you click on `OK'. If you click on `Cancel', your changes will be lost.
Nevertheless, ISE Assembly Manager first checks that your changes are consistent before saving them. In case problems are detected, the list of errors appears at the bottom of the type view. Each error mentions the location of the error, that is "Feature name" (and the original name of the problematic feature) or "Argument name" (with the name of the feature it belongs to and the original name of the argument).

When you click on an error message, the focus will be given to the corresponding text field, inviting you yo modify it.
