The Kernel

The kernel cluster contains basic concepts close to the language level and a set of classes to cover common needs, mainly storage of objects and exception handling.

There are two kinds of classes: interfaces, that are fully deferred and classes that are either partly deferred or totally effective. Indeed, multiple inheritance is not allowed in Eiffel #, but there is no limit in the number of interfaces a class implements. And then, in order to stick to EiffelBase inheritance structure and design, kernel classes have only one base type but inherit as many interfaces as needed.

For now, the kernel cluster contains:

  • Interfaces:
    These classes are fully deferred. They describe features a concrete class must be able to provide when implemented.
    • The CONTAINER interface describes objects that serve to store and access collections of objects. Traversal and access criteria are described by two descendant branches (see Container for more details).
    • BOX and FINITE describe how encapsulated items of Container can be reached (see Box and Finite for more details).
    • TRAVERSABLE and LINEAR how these elements can be traversed, for e.g. in one or two ways (see Traversable and Linear for more details).
    • TABLE interface also describes objects that serve to store and access items, but in this case, each item is associated with a key.
  • Classes:
    These classes are partly or totally implemented. They represent the most commonly used types in Eiffel #.
    • Class ANY: the universal class defining facilities accessible to all other classes. Every developer-defined class is a descendant of this class (see Any for more details).
    • Classes describing the basic types: BOOLEAN, CHARACTER, DOUBLE, INTEGER, INTEGER_8, INTEGER_16, INTEGER_64 and REAL (see respectively Boolean, Character, Double, Integer, Integer_8, Integer_16, Integer_64 and Real for more details).
    • Arrays: class ARRAY (see Array for more details).
    • Strings: class STRING (see String for more details).
    • Pointer: class POINTER (see Pointer for more details).
    • Tuples: class TUPLE (see Tuple for more details).
    • Class HASH_TABLE objects in .NET have exactly the same behavior as in EiffelBase  (see Hash_Table for more details).
    • Classes EXCEPTIONS and EXCEPTION_MANAGER provide control over the exception handling mechanism, in particular for applications that need to handle different types of exception in different ways (see Exceptions and Exception_manager for more details).
    • Class ENUM provides the base class for enumerations (see Enum for more details).
    • Class VALUE_TYPE provides the base class for value types (see Value_type for more details).
