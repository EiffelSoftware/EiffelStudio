Debug/Profile Tab

 

Here is what the debug tab looks like when you have not yet compiled the system you just created.

 

Execution parameters

You can specify the location from where your program will run. By default the location is `your_project_path/EIFGEN/W_code'. Use this option when your application relies on the use of resource files located at a different location.

You can also specify one or more command line arguments. To add one, simply type your command line arguments in the combo box and press enter. To remove one, select it from the combo box and then click on the red cross. To select one, simply choose it from the combo box, and when you click on Apply or Ok they will be automatically taken into account, even before you recompile.

 

Miscellaneous options

  • line number generation when selected will generate in either .NET generation or C generation some extra information that will enable a .NET debugger or a C debugger to step through the Eiffel code rather than MSIL code or C code.
  • call tracing when selected will, in C generation mode only, display on the console all the features that are called during the system execution.
  • profiling when selected will, in C generation mode, turn on the Eiffel profiler during the application execution. The results can be seen at the end of the execution in the profiler window.

Note: The profiler is only available in the Enterprise edition of EiffelStudio.

Debug options

By default all debug clauses are turned off.

The first step to enable them is to select the `Enable debug' check box. Then right-click on the debug clauses you want to enable/diable and a contextual menu will appear to let you do so.

Tip: You can select several debug clauses at a time by using the Ctrl and Shift keys.

Before you compile for the first time your project with an Ace file that has been automatically created, you won't see any debug clauses. They will be automatically added by the compiler at the end of your first compilation. When this is done, you can enable/disable each one of them.
