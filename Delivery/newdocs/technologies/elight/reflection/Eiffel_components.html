2. ISE.Reflection.EiffelComponents.dll


2.1 EIFFEL_ASSEMBLY (EiffelAssembly)

This class gives a representation of a .NET assembly in Eiffel. It has six attributes and the corresponding setters.

assembly_name: STRING
assembly_version: STRING
assembly_culture: STRING
assembly_public_key: STRING
eiffel_cluster_path: STRING
emitter_version_number: STRING

The emitter generates instance of EIFFEL_ASSEMBLY (EiffelAssembly) and passes it as argument to the appropriate generator (depending on the feature you have called), which uses it to generate either Eiffel or XML files. See 5. ISE.Reflection.Emitter.exe for more information.

ISE reflection interface also uses EIFFEL_ASSEMBLY (EiffelAssembly). In fact, when you ask the database for a representation of an assembly, it returns an instance of EIFFEL_ASSEMBLY (EiffelAssembly), which is user-friendlier than the underlying XML file describing assembly. See 3. ISE.Reflection.ReflectionInterface.dll for more information.


2.2 EIFFEL_CLASS (EiffelClass)

This class gives a representation of an Eiffel class generated from a .NET type. It keeps useful information to connect .NET names with Eiffel names, such as dot_net_name. The attributes of class EIFFEL_CLASS (EiffelClass) are listed below:

eiffel_name: STRING
    -- Eiffel name

dot_net_full_name: STRING
    -- .NET full name (i.e. with namespace)

dot_net_simple_name: STRING
    -- .NET simple name

namespace: STRING
    -- Namespace defining current class

parents: SYSTEM_COLLECTIONS_HASHTABLE
    -- Key: Parent Eiffel name
    -- Value: Inheritance clauses
    -- | ARRAY [SYSTEM_COLLECTIONS_ARRAYLIST [STRING]]
    -- (array with rename, undefine, redefine, select and export clauses)

creation_routines: SYSTEM_COLLECTIONS_ARRAYLIST
    -- Creation routines Eiffel names
    -- | SYSTEM_COLLECTIONS_ARRAYLIST [STRING]

initialization_features: SYSTEM_COLLECTIONS_ARRAYLIST
    -- Initialization features
    -- | SYSTEM_COLLECTIONS_ARRAYLIST [EIFFEL_FEATURE]

access_features: SYSTEM_COLLECTIONS_ARRAYLIST
    -- Access features
    -- | SYSTEM_COLLECTIONS_ARRAYLIST [EIFFEL_FEATURE]

element_change_features: SYSTEM_COLLECTIONS_ARRAYLIST
    -- Element change features
    -- | SYSTEM_COLLECTIONS_ARRAYLIST [EIFFEL_FEATURE]

basic_operations: SYSTEM_COLLECTIONS_ARRAYLIST
    -- Basic operations
    -- | SYSTEM_COLLECTIONS_ARRAYLIST [EIFFEL_FEATURE]

unary_operators_features: SYSTEM_COLLECTIONS_ARRAYLIST
    -- Unary operators
    -- | SYSTEM_COLLECTIONS_ARRAYLIST [EIFFEL_FEATURE]

binary_operators_features: SYSTEM_COLLECTIONS_ARRAYLIST
    -- Binary operators
    -- | SYSTEM_COLLECTIONS_ARRAYLIST [EIFFEL_FEATURE]

special_features: SYSTEM_COLLECTIONS_ARRAYLIST
    -- Special features
    -- | SYSTEM_COLLECTIONS_ARRAYLIST [EIFFEL_FEATURE]

implementation_features: SYSTEM_COLLECTIONS_ARRAYLIST
    -- Implementation features
    -- | SYSTEM_COLLECTIONS_ARRAYLIST [EIFFEL_FEATURE]

invariants: SYSTEM_COLLECTIONS_ARRAYLIST
    -- Class invariants
    -- | SYSTEM_COLLECTIONS_ARRAYLIST [ARRAY [STRING]] (Array with invariant tag and invariant text)

is_frozen: BOOLEAN
    -- Is class frozen?

is_expanded: BOOLEAN
    -- Is class expanded?

is_deferred: BOOLEAN
    -- Is class deferred?

create_none: BOOLEAN
    -- Does class have `create{NONE}' creation declaration?

The emitter generates instance of EIFFEL_CLASS (EiffelClass) for each type of the assembly and passes it as argument to the appropriate generator (depending on the feature you have called), which uses it to generate either Eiffel or Xml files. See 5. ISE.Reflection.Emitter.exe for more information.

ISE reflection interface also uses EIFFEL_CLASS (EiffelClass). In fact, when you ask the database for a representation of a type, it returns an instance of EIFFEL_CLASS (EiffelClass), which is user-friendlier than the underlying XML file describing type. See 3. ISE.Reflection.ReflectionInterface.dll for more information.


2.3 EIFFEL_FEATURE (EiffelFeature)

This class gives a representation of an Eiffel feature generated from a .NET member. It keeps useful information to connect .NET feature names with Eiffel names, such as dot_net_name. The attributes of class EIFFEL_FEATURE (EiffelFeature) are listed below:

eiffel_name: STRING
    -- Eiffel name

dot_net_name: STRING
    -- .NET name

arguments: SYSTEM_COLLECTIONS_ARRAYLIST
    -- Feature arguments
    -- | SYSTEM_COLLECTIONS_ARRAYLIST [ARRAY [STRING]]
    -- | Array with argument Eiffel name, .NET name, type (Eiffel name of type) and type full name

return_type: STRING
    -- Eiffel name of feature return type

return_type_full_name: STRING
    -- Full name of feature return type

preconditions: SYSTEM_COLLECTIONS_ARRAYLIST
    -- Feature preconditions
    -- | SYSTEM_COLLECTIONS_ARRAYLIST [ARRAY [STRING]]
    -- | Array with precondition tag and precondition text

postconditions: SYSTEM_COLLECTIONS_ARRAYLIST
    -- Feature postconditions
    -- | SYSTEM_COLLECTIONS_ARRAYLIST [ARRAY [STRING]]
    -- | Array with postcondition tag and postcondition text

comments: SYSTEM_COLLECTIONS_ARRAYLIST
    -- Feature comments
    -- | SYSTEM_COLLECTIONS_ARRAYLIST [STRING]

is_frozen: BOOLEAN
    -- Is feature frozen?

is_static: BOOLEAN
    -- Is feature static?

is_abstract: BOOLEAN
    -- Is feature abstract?

is_method: BOOLEAN
    -- Is feature method?

is_field: BOOLEAN
    -- Is feature a field?

is_creation_routine: BOOLEAN
    -- Is feature a creation routine?

is_prefix: BOOLEAN
    -- Is feature a prefix?

is_infix: BOOLEAN
    -- Is feature an infix?
