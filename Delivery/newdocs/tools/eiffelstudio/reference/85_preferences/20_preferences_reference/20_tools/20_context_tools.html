Context tools

This sub-categories gathers the preferences related to context tools.

There are four of them :
Label in the preferences window Complete description
Share addresses between the editor and the context tool? This boolean preference set whether, by default, the editor and the context tools display information about the same element (Class, Feature or Cluster). The default value is True.
Show ALL classes in the diagram? You can choose not to display some classes in the Diagram Tool. If you set this property to False, you will be given the opportunity to define the list of such classes (see below). The default value is False.
Names of classes that should not appear in generated diagrams. This is the list of classes that will be ignored in the Diagram tool if the previous preferences is set to False. By default, this list includes : INTEGER, BOOLEAN, STRING, CHARACTER, REAL and DOUBLE.
Order used to sort feature clauses in class views. This sets the order of appearance of feature clauses in class text when it is displayed in all views but the basic text one (i.e. Clickable, Flat, Contract or Flat Contract views).
Show all callers in `callers' view. When the Feature Tool is centered on a feature, you can see its callers by requesting the `callers' view. If this property is set to True, this of callers will include calls to descendant versions of the feature. Otherwise, it will not. The default value is True.
Show syntactical suppliers (as opposed to actual suppliers) in the `suppliers' view. There are two types of suppliers for a given class. On one hand, there are the syntactical suppliers, i.e. the ones the name of which appear in the class text. On the other hand, there are the actual suppliers, i.e. the ones features of which are used in the class. This preference allows you to choose what type you want to be displayed when you select the `suppliers' view in the class tool. If it is True, syntactical suppliers will be shown. The default value is False.
Indexing items that need not be included in class views. This is a list of items in the indexing clause that will be hidden in Clickable, Flat, Contract and Flat Contract views of classes. By default, it includes "revision", "date" and "status".
Class view that is selected by default. This is the number of the view that be used by default in the Class tool. To find the number associated with a view, refer to the table below. The default value is 5.
Feature view that is selected by default. This is the number of the view that be used by default in the Feature tool. To find the number associated with a view, refer to the table below. The default value is 2.
