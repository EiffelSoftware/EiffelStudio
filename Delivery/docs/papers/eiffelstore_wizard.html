EiffelStore Wizards (Beta Version)


These Wizards will help you to create an Eiffel Project using EiffelStore.

  • First you will have to provide the information in order to connect to a Database.
  • Then, the wizard will ask you the Tables you will need in your Project.
  • Finally, it will generate an Eiffel Example and compile it.
  • You can then modify the example to create your own Application.

For the moment, they have been compiled only for Windows using either Oracle or ODBC as a back end database. You can download them from our ftp site at:
  • ftp://ftp.eiffel.com/pub/download/windows/4-5/store_wizard_oracle.zip.
  • ftp://ftp.eiffel.com/pub/download/windows/4-5/store_wizard_odbc.zip.


Installation:


Unzip the file EiffelStore_Wizard.zip. Be sure to use folder names when unzipping. This will create a subdirectory 'Store_wizard_odbc'. In this directory there should be 3 other directories (bmp,exe,ressources).

Then, execute launch.bat

However, you can directly run the exe file in the directory exe, with as parameter the path where you have unzipped the wizard.

Example: store_wizard_odbc c:\store_wizard_odbc

Generated Files:

The Wizard will generate an Eiffel example using your database and your tables.

The generated classes are:

  • One class for each chosen table. 

These classes will be use to retrieve the data from the database.
There is a mapping from your RDBMS database to these Eiffel classes.
Thus, each classes will have the same name as the tables in your database and for each their attributes there will be an Eiffel attribute with the same name and type.

  • The class DATABASE_MANAGER which help you to do any queries on the database
  • The classes: DB_ACTION, DB_ACTION_DYN, REPOSITORIES and DB_SHARED which are used by the DATABASE_MANAGER classes.

These are the core classes to access and modify the Database.

  • The classes ESTORE_EXAMPLE and ESTORE_ROOT

These class are the main classes of the Example.
You can modify and add any queries by editing ESTORE_EXAMPLE.

In the Directory resources of the EiffelStore Wizard, you can find the skeletons of the Generated files.
You can modify them to fit with your project.

Known Limitations:

  • An Ace file in generated , and so the link to ODBC32.lib is: "$EIFFEL4\library\store\spec\$(COMPILER)\lib\odbc32.lib". So you should copy this file in this previous directory to avoid any problems. Or you can change the link in the generated ace file.

  • You must choose at least one Table before doing the generation.

  • For now there are not any differences in the generation when you choose to generate a new project, or an existing project.

  • The wizard has been developed using Vision2. However, there are still a few limitations with vision2 on EV_TEXT. That's why the fonts are not so nice for now. Also if you are using the large fonts, you won't read all the comments of the wizard.

  • After the generation, the compilation and ebench are launched. However, if there was an other project in the directory, the compilation can't be launched. So the Directory is deleted before the generation. If you don't have the permission to delete some files, the compilation won't be launched.

  • If you are using MS SQL Server you should choose the check box : specific Tables/Views. Indeed, you shouldn't try to generate the System tables as you won't be able to make any request on some of these tables.

  • If you have a $ in the name of your Table, for now EiffelStore doesn't support it. So the compilation won't work as an Eiffel Class can't have a $ in its name.

  • The chosen table must have at least one Column.
