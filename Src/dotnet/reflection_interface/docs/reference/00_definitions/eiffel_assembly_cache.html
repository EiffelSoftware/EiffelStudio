The Eiffel Assembly Cache



The Eiffel Assembly Cache is a repository where the compiler saves the required information to be able to associate Eiffel names with .NET names.

It also keeps tracks of the changes you make when editing types via the type view: the Eiffel Assembly Cache is updated (i.e. existing names replaced by the new ones) once ISE Assembly Manager has checked the changes consistency.
ISE Assembly Manager is notified whenever the Eiffel Assembly Cache is modified, so that it always displays the relevant names.

Note that the Eiffel Assembly Cache is self-sufficient: When you launch the Eiffel code generation from ISE Assembly Manager (via the Eiffel generation dialog), the generator finds all needed information in the Eiffel Assembly Cache. There is no need to explore the assembly metadata anymore, which results in noteworthy gain of time.


The Eiffel Assembly Cache is located in "$ISE_EIFFEL"\dotnet\assemblies", where $ISE_EIFFEL is the path to your Eiffel delivery. If you explore it, you will see that information is saved in separate folders, each folder corresponding to one imported assembly.
