Measuring

Now that we have defined all basic notions such as metrics. metric hierarchy and scopes, let us see how the metric tool can help in evaluating a project.

Many commands are provided in the tool to carry out measures and handle them. Indeed, it is possible to calculate metrics, save measures in order to retrieve them later on, request details for a given measure when available and eventually delete a set of measures.

To perform a measure, you will simply choose a metric and a scope, and request the computation, thanks to the "calculate" button. this operation allows to evaluate the selected metric over the selected scope. Measure is displayed in the provided text field, but it is not yet saved.

To multi column list displays the recorded measures. To save a measure, click on the "Add & save" button. The last calculated measure appears then in the multi column list and is being saved in a file auto generated by the interface. It is located in the Metric subfolder of the current project location.

For "big" scopes (in the sense of the order relation previously defined, that is including smaller scopes) details are sometimes available regarding the evaluated metric, and then the "detail" button is sensitive. It displays measures resulting from applying the requested metric over all smaller scopes.

It is possible to remove some measures after having saved them thanks to the "delete" button. Measures will be removed both from the multi column list and the file they were saved in.

Eventually, after having recompiled a project, the recorded measures may be no longer accurate. The icon replaces to inform user that measures have been calculated before recompilation. Updating measures is possible by right clicking on the multi column list and choosing an item from the popped up menu. Measures will be updated both in the multi column list and the file they were saved in.
