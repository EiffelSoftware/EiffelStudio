Project settings window

 

This is where global settings regarding your project are going to be made. In here you can specify:

  • the clusters that contain your classes
  • which assertion level you want to check
  • which debug clauses will be activated
  • the command line arguments and the current working directory when you debug your application
  • which C externals your project needs
  • the compiler options concerning optimization
  • advanced options such as multithreading

 

The window is organized in six tabs:

  1. General
  2. Debug/Profile
  3. Clusters
  4. .NET
  5. Externals
  6. Advanced

The .NET tabs is only enabled on the Windows platform.

 

Most of the changes that can be made here are taken into account when you next recompile your project. Instead of automatically performing the compilation each time you click Ok or Apply we decided to save the modifications and let the user decide when to recompile. At the moment, only the specification of the command line arguments and of the current working directory for debugging do not require a recompilation of your system.
