Comparing to an archive

The main goal of archives is comparing archives one to another. In order to compare your project to an archived file, you must have clicked on in the "metric" tab of the context tool.

The goal is to load the archive of a (preferably different) project, to compare the archived measures to the system's. You can either select a file that is located on your machine or a remote one, using a URL address.

  1. In the "Set reference archive" frame, select the comparison mode. Comparison can be displayed in one of the following mode:
    - Ratio of current measures to the archived one.
    - Ratio of the difference between current and archived measures to current measures.
    - Value of the archived measures.


If you want to access a remote archive:

  1. Enter the URL address in the " Current archive" field. Address must include either a http or a ftp protocol.
  2. Press the carriage return (enter) key.

If you already have the archive file on your machine:

  1. Click on "Browse".
  2. Select the archive file you want your system to be compared to.

In both cases, exit the main interface (on clicking "OK") to make your comparison effective. Comparison results are displayed in the last column of the multi column list in the "Metric" tab of the context tool, regarding the comparison mode you have previously chosen.

Note: "Current measures" are the result of evaluating available metrics over the current system scope, when possible.
