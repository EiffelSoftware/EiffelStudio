Evaluation tool

The expression evaluation tool's main part is the list of expressions that should be evaluated.

Its first column gives the context in which the expression is evaluated. Its second column gives the expression itself, as it was entered by the user. The last column gives the result of the expression.

Tip: If the expression returns a non-Void object, then the corresponding row in the list is pickable, and can be dropped for example in the object tool to inspect its state.

On top of the list, three buttons can be found.

  • pops up the expression definition dialog, and can be used to define new expressions.
  • edits a currently selected expression (it is not possible to change the context of an expression, only the expression itself).
    The associated key shortcut is F2.
  • removes the selected expressions from the list.
    The associated key shortcut is Del.

Tip: Dropping an object onto this list lets you define quickly an expression whose context is the dropped object. This avoids having to type object addresses manually.
