Vision2 Events

This cluster contains classes for event handling within Vision2.

What is an event?

An event is considered to be an external action that occurs during a program's execution. Correctly dealing with events is an important part of developing a Vision2 application. For example, if a user selects a menu item from a menu, the application needs to be able to respond appropriately to this event and perform the necessary processing.

How do I connect to an event?

When an event occurs, the corresponding action sequence is called. All Vision2 action sequences inherit EV_ACTION_SEQUENCE and when it is called, all of the agents held within are fired, thus calling all of the procedures represented by the agents. The signature of any agent that you place in an action sequence must conform to those of the action sequences actual generic parameter.

There are two different types of classes contained within this cluster, identified by their names:
  • classes named ev_*_action_sequence.
    These classes represent different types of action_sequence, distinguished by the parameters that their items can take. For example, one of the most widely used is EV_NOTIFY_ACTION_SEQUENCE whose items take no arguments. All these classes inherit EV_ACTION_SEQUENCE
  • classes named ev_*_action_sequences.
    These classes contain one or more action sequences applicable to a certain context. For example, EV_BUTTON_ACTION_SEQUENCES contains one action sequence, select_actions, which is fired whenever a button is selected in the GUI.
