Release Notes

We hope that you will enjoy the the wizard, which greatly facilitates the writing and debugging of COM applications, using Eiffel on the server side, the client side or both.

Here are some notes applying to the current version:

  1. Client
    • You should always access dual interfaces through IDisptch::Invoke instead of direct virtual table access.
    • Memory for [in] parameters may cause memory leaks insome special cases (nested pointers).
  2. Server: Automation is not properly supported on the server side.
  3. Eiffel to C type mapping: the following COM types are not supported: SAFEARRAY and its VARIANT.

If you encounter any other problems please do not hesitate to contact through the support web page http://support.eiffel.com or through the Talkitover group http://www.talkitover.com/eiffel/com.

There will be further intermediate releases of the EiffelCOM Wizard.

Each will be announced on the Talkitover group http://www.talkitover.com/eiffel/com.
