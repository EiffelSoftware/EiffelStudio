1. Introduction

ISE reflection system is composed of four components interacting with each other: `Emitter.exe', `ISE.EiffelComponents.dll', `ISE.CodeGenerator.dll' and `ISE.ReflectionInterface.dll'.

  • `Emitter.exe' enables you to generate Xml files and Eiffel classes from a .NET assembly. This relies on the three other components described below. You can call `Emitter.exe' directly.


  • `ISE.EiffelComponents.dll' is used internally by `Emitter.exe', `ISE.CodeGenerator.dll' and `ISE.ReflectionInterface.dll'. It provides a representation of an Eiffel class, an Eiffel feature and an Eiffel assembly.


  • `ISE.CodeGenerator.dll' is used by `Emitter.exe' to generate Xml files and Eiffel classes from Eiffel components. It is also used by `ISE.Reflection.Interface.dll' to retrieve Eiffel components from Xml files.


  • `ISE.Reflection.Interface.dll' is aimed at connecting .NET names with corresponding Eiffel names by using XML files. It is a kind of database where you put XML files describing .NET types and you retrieve a more user-friendly type description. (Only shared assemblies can be stored in this database.) You can use it directly to retrieve information from the database.

Since all other components rely on it, this specification describes `ISE.EiffelComponents.dll' first.
