Generation Tab

 

This tab contains information about the available code generation options.

Here is the appearence of the Generation tab with default settings for a Build project:

The options available in this tab are as follows:

  • Attribute declarations
    • Local - All attributes (corresponding to widgets and items generated by EiffelBuild) are declared as local variables.
    • Exported - All attributes are declared in the body of genrated classes, and are therefore exported and available to clients.
    • Export only named - Only those attribtues that have been named are exported, while the others are declared as locals. In most situations, those widgets that are not named, have no further use except in the construction of the interface, while those named have been done so, as they will be used again.
    • Grouped - Should different objects of same type be declared individually, or grouped?

        button1: EV_BUTTON
        button2: EV_BUTTON
        button3: EV_BUTTON
      or
         button1, button2, button3: EV_BUTTON
  • Attributes declared as locals

    • Should attribute declarations be local to initialization feature, or attributes of the class?

  • Generate debugging information

    • If selected, then for every feature connected to an event, code will be generated which outputs to the console, the name of the feature, and the values of all arguments.

  • Client of windows

    • If selected, then every generated class representing a window or dialog will use the window as a client. If not selected, the generated classes inherit the window classes directly.
